**Frontmatter fields:**
  name: gsd:iterate
  description: Run the full code-first iteration loop - extract tags, generate plan, approve, execute (Code-First fork)
  argument-hint: "[--non-interactive] [--verify] [--annotate]"
  allowed-tools: Read, Write, Edit, Bash, Task, Glob, Grep

**Objective section:**
Explain that iterate orchestrates the complete code-first development loop. It extracts @gsd-tags from annotated code, spawns gsd-code-planner to generate a plan from CODE-INVENTORY.md, pauses for human approval of the plan, then spawns gsd-arc-executor to implement it. Optional flags:
- `--non-interactive` -- auto-approves the plan (for CI/headless pipelines, per D-03/ITER-03)
- `--verify` -- runs verification after executor completes (per D-04/ITER-02)
- `--annotate` -- refreshes @gsd-tags after executor completes (per D-05/ITER-02)

**Context section:**
  $ARGUMENTS
  @.planning/PROJECT.md
  @.planning/REQUIREMENTS.md
  @.planning/ROADMAP.md

**Process section** -- 6 steps, chained sequentially per D-06. Each step checks for failure and stops with error report if any step fails:

**Step 1 -- Show current mode and run extract-tags** (per D-10):
Read current mode via bash: `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" config-get default_phase_mode`
Display: "Current mode: {mode}"
Then run extract-tags via bash:
  `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" extract-tags --format md --output .planning/prototype/CODE-INVENTORY.md`
If extract-tags fails (exit code != 0), stop and report: "iterate failed at step 1: extract-tags error."

**Step 2 -- Spawn gsd-code-planner:**
Spawn `gsd-code-planner` via the Task tool. The agent reads `.planning/prototype/CODE-INVENTORY.md` (produced by step 1) and the source code @gsd-tags to generate a plan. Wait for completion. If the agent fails, stop and report: "iterate failed at step 2: code-planner error."

**Step 3 -- Approval gate** (per D-02, D-03):
Check if `--non-interactive` is in `$ARGUMENTS`.
- If `--non-interactive` is present: skip approval, proceed to step 4. Log: "Auto-approving plan (--non-interactive mode)."
- If `--non-interactive` is NOT present: Read the generated plan and present a summary to the user. Ask the user: "Review the plan above. Approve execution? [yes/no]". If the user responds with anything other than clear approval (yes, y, approve), stop and report: "iterate stopped: plan not approved."

**Step 4 -- Spawn executor:**
Check if `arc.enabled` is true in config via bash: `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" config-get arc.enabled`
If true, spawn `gsd-arc-executor` via Task tool. If false or not set, spawn `gsd-executor` via Task tool.
Pass the plan path as context. Wait for completion. If the agent fails, stop and report: "iterate failed at step 4: executor error."

**Step 5 -- Post-execution flags** (per D-04, D-05):
- If `--verify` is in `$ARGUMENTS`: run `/gsd:verify-work`
- If `--annotate` is in `$ARGUMENTS`: run extract-tags again via bash:
  `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" extract-tags --format md --output .planning/prototype/CODE-INVENTORY.md`

**Step 6 -- Show summary:**
Display: steps completed, plan path, whether verification ran, whether re-annotation ran, and any skipped steps.
