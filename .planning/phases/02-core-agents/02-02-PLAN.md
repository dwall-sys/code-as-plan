Execution flow (`<execution_flow>` XML tag) with 5 numbered steps:

Step 1 `load_inventory`: Read `.planning/prototype/CODE-INVENTORY.md` as primary input. Parse the tag groups: count @gsd-todo tags (these become tasks), read @gsd-context tags (background), read @gsd-decision tags (background), read @gsd-constraint tags (hard limits), read @gsd-risk tags (special handling), read @gsd-api tags (interface contracts), read @gsd-pattern tags (patterns to follow), read @gsd-ref tags (requirement traceability).

Step 2 `scan_source`: Run a fresh tag scan for supplementary detail:
  node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" extract-tags --format json
Use the JSON output to get exact file paths, line numbers, and full tag text. Read the actual source files around tagged locations (5-10 lines of context) to understand the code structure.

Step 3 `plan_tasks`: For each @gsd-todo, create a task. Task name: derived from the todo description. Files: from @gsd-ref metadata, or inferred from the file where the todo lives. Action: what to implement, incorporating any @gsd-constraint tags from the same file/module. Done when: success criteria derived from the todo description and any related @gsd-api contracts. Order tasks by dependency (if task B references a type created by task A, A comes first).

Step 4 `write_plan`: Write the plan to `.planning/prototype/CODE-PLAN.md` using this exact format:
  Heading: # Code Plan: [Description derived from @gsd-context tags]
  Section "## Context": 2-5 sentences summarizing architectural context from @gsd-context and @gsd-decision tags
  Subsection "**Constraints:**": bullet list from @gsd-constraint tags
  Subsection "**Risks:**": bullet list from @gsd-risk tags, or "None identified" if no risk tags
  Section "## Tasks": each task as ### Task N: [Name from @gsd-todo], with **Files:** (exact paths), **Action:** (specific instructions), **Done when:** (measurable criteria)
  Section "## Success Criteria": checkbox list with overall criteria, "All @gsd-todo items addressed", and requirement-specific criteria from @gsd-ref tags

Step 5 `report`: Summarize number of tasks generated, files targeted, path to CODE-PLAN.md.

Constraints block (`<constraints>` XML tag at end of agent prompt):

2. NEVER include research or discovery sections -- annotations ARE the research
3. Plans must be compact enough for a single executor pass (target: 2-8 tasks)
4. If more than 10 @gsd-todo tags exist, group related todos into combined tasks
5. Every task must have Files, Action, and Done-when fields
6. Always read CODE-INVENTORY.md before planning -- do not plan from scratch
7. Reference requirement IDs from @gsd-ref tags in success criteria
8. Use Write tool for all file creation, never heredoc via Bash
9. Write plan to .planning/prototype/CODE-PLAN.md
